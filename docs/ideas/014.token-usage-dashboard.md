# Token 用量看板

- **ID**: ideas-014
- **状态**: proposed
- **优先级**: P1
- **来源**: `docs/ROADMAP.md` · v0.2.5+ "UI / 可观测性"
- **创建日期**: 2026-05-24

## 背景
用户反馈优先级偏高:目前 token 消耗黑盒,看不到"今天主/辅 LLM 各花了多少",无法为调参 / 换模型提供依据。在多 Provider(Qwen / DeepSeek)接入后,这条价值立刻凸显。

## 提案
- **采集**:在 `QwenProvider.chat` / `QwenEmbeddingProvider.embed` 的 stream `finish` / 响应处拿 `usage.{prompt,completion,reasoning}Tokens`。
- **聚合维度**:`provider-kind × model-id × 日期(UTC+8)` → 写入新表 `token_usage`(Dexie schema 升级)。
- **角色标签**:bootstrap 装配 Provider 实例时打"主 / 辅 / embedding / reflection"标签,便于追溯哪类调用最耗。
- **展示**(配置页新增"用量" Tab):
  - 今日:主 / 辅 / embedding 各模型柱状图 + 总数
  - 最近 7 天趋势折线
  - 累计(若用户填了单价显示金额)
- **保留期**:本地 90 天,启动时清理过期。

## 验收标准
- [ ] 新表 + 写入路径 + Tab 展示三段贯通。
- [ ] 反思 Job 调用打 "reflection" 来源标签。
- [ ] 数据只在本地 IDB,不上传。
- [ ] 单测覆盖聚合 + 过期清理。

## 备注
- 千问 usage 字段在 AI SDK stream 的 `finish` part 里,`normalizer.ts` 已归一化为 `ChatChunk.usage`,可直接消费。
- 与 ideas-002(subAgent)天然协同 — 子 agent 的 token 预算隔离需要本看板可视化。
