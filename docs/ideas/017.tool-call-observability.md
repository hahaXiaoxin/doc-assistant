# UI 层 tool-call 可观测性

- **ID**: ideas-017
- **状态**: proposed
- **优先级**: P2
- **来源**: `docs/ROADMAP.md` · v0.2.5+ "UI / 可观测性"(源于 TROUBLESHOOTING §10 启示)
- **创建日期**: 2026-05-24

## 背景
当前 `useStreamingChat.applyChunk` 对 `tool-call` / `tool-result` 故意不渲染(MVP 注释为证),用户在 UI 看不到工具调用痕迹。TROUBLESHOOTING §10 已记录此模式让"Persona 语义错位"这类设计 bug 静默存在了很久才被发现。

## 提案
- assistant 消息加"已调用 N 个工具"小徽章(最低限度可观测性,不打扰主消息流)。
- 点击展开:逐 tool 显示 name / 参数 / 结果(JSON 折叠);失败 tool 标红。
- 与 ideas-014 用量看板互补 — 一个看"做了什么",一个看"花了多少"。

## 验收标准
- [ ] 徽章默认折叠,展开有完整 tool-call 流。
- [ ] 失败 tool 视觉与正常区分。
- [ ] 历史消息回看时徽章可还原(从 `episodes_msg` 持久化的 tool 元信息恢复)。

## 备注
- 与 ideas-002(subAgent UI)同源 — 子 agent 折叠气泡可复用同一组件。
- TROUBLESHOOTING §10 启示:**"tool description 隐含主体必须显式写出来"** + UI 默认隐藏 tool 让设计 bug 静默 — 本条是后者的对症。
