# 引用标签(Ref Chip)富媒体扩展

- **ID**: ideas-004
- **状态**: proposed
- **优先级**: P1
- **来源**: `docs/backlog.md` · B-005
- **创建日期**: 2026-05-24

## 背景
当前划词引用在输入框是 chip,提交后存为伪 HTML `<ref>` 字符串,在气泡里被 `rehype-sanitize` 默认白名单当未知标签降级为纯文本。同时 payload 仅承载文本,后续要扩展图片 / DOM 快照 / PDF 片段时缺数据模型。

## 提案
把"引用"从"LLM 消费的序列化字符串"升级为"前端可识别的一等对象"。
- 数据模型:`ContentRef`(判别联合,至少 `kind: 'text' | 'image'`,留位 `dom-snapshot / pdf-region`),UI 组件 `RefChip`。
- 文本型 MVP:气泡里 chip 完整可用(展示 + hover 原文 + 点击跳锚点)。
- 图片型占位:有一条端到端 happy path(配置页贴图即可),证明扩展位不是 PPT。
- 历史消息向后兼容:老 `<ref>` 字符串就地升级或兼容渲染,不能丢消息、不能降级纯文本。
- LLM 侧契约保留:`userInput` 仍含 `<ref>` 或等价语义,`referenceTagSource` 注入段不变。

## 验收标准
- [ ] 输入框与气泡中 chip 视觉一致(颜色 / hover / 截断)。
- [ ] markdown(代码块/列表/表格)与 chip 共存,互不打断结构。
- [ ] `rehype-sanitize` 白名单不无差别放开,新自定义元素显式注册。
- [ ] `serializer.test.ts` / `MessageBubble` 测试新增"含 ref 的消息双向往返"用例。

## 备注
- 与 Provider / ChatChunk 弱耦合,理论上不动 Provider 层。
- 与 ideas-001 正交,但建议先于其落地,避免多智能体引入后引用场景扩散再返工。
- 候选命名 `ContentRef` + `RefChip`(数据模型 + UI 组件,最终以 developer 实装时为准)。
