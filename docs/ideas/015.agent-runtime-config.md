# 集中化 AgentRuntimeConfig(代码配置文件)

- **ID**: ideas-015
- **状态**: proposed
- **优先级**: P1(基础设施治理)
- **来源**: `docs/ROADMAP.md` · v0.2.5+ "工程治理"
- **创建日期**: 2026-05-24

## 背景
Agent 优先级、`maxTurn`、反思 Job 间隔、召回粗判触发词、tool description 措辞片段等"运行时可调参数"散落在 `packages/shared/src/config.ts`、`packages/agent/src/loop.ts`、`packages/agent/src/reflection/*`、`packages/tools/src/definitions/**` 等多处。每次"把 maxTurn 8 调到 12"或"调一句 tool description"都要改代码 + 重新构建,改动分散、容易遗漏、难以可视化对比。

## 提案
新增 `packages/shared/src/agent-config.ts`(或 `packages/shared/config/agent.ts`)作为唯一可信源(`AgentRuntimeConfig`),首批抽取 ≥ 5 类参数:
- 各 agent 优先级
- 主循环 / 各 agent 的 `maxTurn`
- aux LLM `temperature / topP / maxTokens`
- 反思 Job `scanIntervalMinutes / retryLimit / batchSize`
- 召回链路 keyword 触发集合 / intent 阈值 / 向量 topK / 邻居拼接窗口
- 各 tool description 中"可外置"的文案片段
- SessionTopic `shouldIdentify()` 触发周期 + 漂移阈值

设计:`DEFAULT` → `profile 覆盖` → `用户 chrome.storage 覆盖` 三层 merge。bootstrap 时 `loadAgentRuntimeConfig()` 注入消费方。`DEFAULT` 取值与现有魔法常量等价(零行为变化)。

## 验收标准
- [ ] 至少 5 类参数被抽出。
- [ ] 全部消费方走 config,旧位置 grep 不到遗留魔法常量。
- [ ] 单测:DEFAULT 冻结、merge 合并、扩展字段、边界值 clamp(如 `maxTurn<=0`)。
- [ ] `docs/` 内附配置速查表(名称 / 默认值 / 取值范围 / 影响面)。
- [ ] (加分)配置页"调试" Tab 只读视图展示当前生效 JSON。

## 范围边界(本次不做)
- ❌ GUI 编辑器
- ❌ 多 profile 切换 UI
- ❌ system prompt 全文搬进 config(只外置可调片段)
- ❌ 配置热更新(MVP 仅 bootstrap-time 加载)

## 备注
- ⚠️ 不要把 API Key / baseURL / model id 塞进来 — 这些是"用户配置"不是"超参",职责要分清。
- ⚠️ ideas-006 的相似度阈值、ideas-009 的缓存时长等都建议归口此处。
- 不解决用户可感知痛点,但显著降低后续所有 prompt / agent 调优成本,**越晚做越贵**。建议 v0.6 之前或并行完成。
