# WebSearch · 迭代式联网检索

- **ID**: ideas-010
- **状态**: proposed
- **优先级**: P2
- **来源**: `docs/backlog.md` · B-009-b
- **创建日期**: 2026-05-24

## 背景
WebSearch 不是"搜一次返回 N 条"的一次性 tool,而是 Claude / Perplexity / OpenAI Deep Research 的**搜 → 拉 → 判 → 不够则换关键词再搜 / 深入二级页面 → 直至产出带引用的答案**范式。这本质需要**独立上下文、能自循环、不污染主对话**的子 agent — 因此本条**强依赖 ideas-002 subAgent**;没有 subAgent 只能退化成"搜一次返回列表"的弱版。

## 提案
- tool `web_search(query)` 返回结构化结果(title + url + snippet),由子 agent 配合 `web_fetch` 迭代深入。
- **三选一搜索路径**(待调研拍板):
  1. 直接接搜索引擎 API(Google CSE / Bing / Brave / Tavily / Serper)— 用户填 API key
  2. 爬搜索引擎结果页(法律 / 反爬 / 稳定性都不友好,不推荐)
  3. **复用用户浏览器已开的搜索引擎 Tab**(扩展独有路径,需模拟用户操作)
- **子 agent 终止条件**(留 ideas-002 落地后定):最大迭代轮数 / token 预算 / 相关性阈值 / 无增益收敛 / 用户中断,任一命中即停。
- 调研建议起点:**Tavily(为 LLM 设计) + SearxNG(自托管零 API key)**双路径对比。

## 验收标准
- [ ] 至少打通一个搜索引擎渠道,返回结构化结果列表。
- [ ] 子 agent 迭代循环可用:搜 → fetch → 判 → 再搜,产出带引用的答案。
- [ ] 终止条件实装,符合 ideas-002 子 agent token 预算红线。
- [ ] 配置页"搜索引擎"槽(API key / 引擎选择)。
