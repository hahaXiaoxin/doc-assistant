# OCR / 多模态识图

- **ID**: ideas-012
- **状态**: proposed
- **优先级**: P2(版本主题:v0.7 / Phase 3-a)
- **来源**: `docs/ROADMAP.md` · §3
- **创建日期**: 2026-05-24

## 背景
让助手能处理 Canvas 渲染的内容(数学公式、图表、扫描 PDF 文本)。`packages/tools/src/ocr/interface.ts` 已定义 `OCRStrategy` 骨架。Tools 层当前 ESLint 禁 `tesseract.js`,本期解禁。

## 提案
策略模式两条:
1. **TesseractOCRStrategy**(第一步,解除 ESLint 约束)
2. **MultimodalLLMOCRStrategy**(第二步,调千问 `qwen-vl-plus` 等多模态模型)
3. 兜底:所有策略失败返回 `{ text: '', confidence: 0 }` 不抛错。

新增 LLM tool:`capture_canvas` / `capture_region` / `recognize_text`。

## 验收标准
- [ ] `OCRStrategy` 接口具体实现 ≥ 1 套。
- [ ] 三个新 tool 可用,流式 / 错误兜底正确。
- [ ] 配置页提供识别开关与"显式告知用户"提示。
- [ ] URL 黑名单(身份证 / 验证码场景)红线落地。

## 备注
- 合规红线:识别前显式告知;禁止自动识别验证码 / 身份证。
- 与 ideas-004(Ref Chip 富媒体)弱协同 — 截图引用与 OCR 输入是同根抽象。
