# Persona 类调用支持独立 Provider

- **ID**: ideas-007
- **状态**: proposed
- **优先级**: P1
- **来源**: `docs/backlog.md` · B-007-C
- **创建日期**: 2026-05-24

## 背景
`ReflectionRunnerDeps.aux` 一把抓"SessionTopic / Intent 精判 / 反思摘要 / Persona 抽取"等所有辅助任务,配置粒度一刀切。但 persona 抽取/整合/相似度判断对判别力要求显著高于其它 aux 任务,被均码模型锁住天花板;ideas-005 的 prompt 边界效果也无法跑到更强模型上。

## 提案
- `ReflectionRunnerDeps` 拆分或扩展,新增可选 `personaProvider?: LLMProvider`,未提供时回退 `aux`(向后兼容)。
- 配置页 "Provider" 分组新增可选 `persona` 配置槽(允许为空 = 复用 auxiliary),与 `main / auxiliary / embedding` 并列。
- README + ROADMAP §6 标注"persona 类任务可独立配置模型"。
- 推荐配置注脚:DeepSeek R1(推理型,判别力更强)是 persona 抽取/整合的性价比最优。

## 验收标准
- [ ] 旧用户未设置 `persona` 时行为与现状 100% 等价。
- [ ] 配置 `persona` Provider 后,反思 runner 中 persona 三类调用走新 provider。
- [ ] 文档 + 推荐配置注脚同步。

## 备注
- 弱依赖 B-001(已落地的 DeepSeek Provider)— 配合 R1 收益最大;
- 与 ideas-005 共同放大效果(边界 prompt × 更强模型)。
