# 开源仓库基础设施

- **ID**: ideas-008
- **状态**: proposed
- **优先级**: P2
- **来源**: `docs/backlog.md` · B-008
- **创建日期**: 2026-05-24

## 背景
2026-05-06 根 README 已重写为开源项目形态,但配套琐事尚未落地:License 待项目所有者拍板、CONTRIBUTING / Issue·PR 模板缺失、README 截图为 `<!-- TODO -->` 占位、CI 徽章为 `license=TBD` 占位。这些项不影响功能,但影响首个公开 Release 的形象。

## 提案 / 子项清单
- `LICENSE` 文件 + README License 章节对齐(MIT / Apache-2.0 二选一,**唯一硬前置:项目所有者决策**)。
- `CONTRIBUTING.md`(分支策略、PR/Issue 模板要求、代码风格指回 ESLint/Prettier/tsc -b)。
- `.github/ISSUE_TEMPLATE/*.yml` + `.github/PULL_REQUEST_TEMPLATE.md`(bug / feature / question 三类起步)。
- 至少 5 张核心截图(侧边对话框 / 划词引用 / Persona 审核 / 记忆浏览器 / 配置页)放 `docs/assets/` 或 `docs/images/`。
- README 顶部徽章替换为 shields.io 真实数据源(license / version / CI / PRs Welcome)。
- (可选)Code of Conduct(社区规模起来后补 Contributor Covenant)。

## 验收标准
- [ ] 根目录有 `LICENSE`。
- [ ] 根目录有 `CONTRIBUTING.md`,README "贡献"章节链向它。
- [ ] `.github/ISSUE_TEMPLATE/` 与 `.github/PULL_REQUEST_TEMPLATE.md` 就位。
- [ ] README 5 张截图替换 `<!-- TODO -->`;徽章数据源真实。

## 备注
- 子项之间互相独立,可以分批合入。
- License 不定版前不建议正式打 Release tag。
