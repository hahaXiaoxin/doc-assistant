# subAgent · 子智能体委派

- **ID**: ideas-002
- **状态**: proposed
- **优先级**: P2
- **来源**: `docs/backlog.md` · B-003
- **创建日期**: 2026-05-24

## 背景
`packages/agent/src/orchestrator.ts` 已有多 agent 骨架,但生产路径只跑一个 ChatAgent。"主 agent 规划 + 子 agent 干活"的范式(Claude Code / Cursor / Devin 已验证)缺位,导致复杂任务上下文过载、tool 调用链过长、推理深度不够。

## 提案
- 任一自定义智能体可声明 `invocable-as-subagent: true`。
- 主 agent 增加 tool `delegate_to_subagent(name, task, contextHints)`:子 agent 默认不继承父 ChatHistory,只拿 SystemPrompt + 任务 + hints。
- 子 agent 独立 ContextSource 组装、独立 tool-call loop、独立 maxTurn(建议 ≤ 父的 1/2)、独立 token 预算。
- Persona / Episodic 子 agent 默认可读不可写。
- 返回结构化结果给父 agent,父 agent 作为 tool result 继续推进。
- UI 折叠气泡显示"调用子智能体 X 中...",可展开审计完整对话。

## 验收标准
- [ ] tool `delegate_to_subagent` 可用,父子上下文隔离。
- [ ] 不支持嵌套(MVP 明确关掉);单次会话最多并行 N=3 个子 agent。
- [ ] 子 agent token 用量计入用量看板。
- [ ] 单测覆盖正常委派 / 子 agent 失败兜底 / 并发隔离 / 嵌套拒绝。
- [ ] sidebar UI 折叠气泡可展开。
