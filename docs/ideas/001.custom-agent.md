# 自定义智能体(领域专家)

- **ID**: ideas-001
- **状态**: proposed
- **优先级**: P1
- **来源**: `docs/backlog.md` · B-002
- **创建日期**: 2026-05-24

## 背景
当前 ContextSource 组合(SystemPrompt / Persona / SessionTopic / WorkingMemory / ChatHistory / RelevantMemory)对所有用户、所有场景一视同仁;但读论文、读源码、查信息、整理协作文档需要的"人设 + 工具集 + 关注点"差异巨大。Persona 机制是单一维度的"长期指令 / 用户画像",承载不了"领域人设 + 工具包 + 提示词片段"的整体切换。

## 提案
把"智能体"提升为一等公民的用户可配置对象:
- 配置页新增"智能体" Tab,支持创建/编辑/删除/排序,含名称/图标/简述/匹配规则(域名/URL 正则/关键词)/系统提示词/启用 tool 子集/可选专属 Provider。
- Schema 预留 `type: 'interactive' | 'background'`、`trigger: domain | manual | dom-event`、`outputChannel: 'main-stream' | 'side-channel'`、`defaultEnabled` 四个字段(为 v0.8 CheckerAgent 让位,作为 background 实例承接,不再单独演化)。
- sidebar 顶部"当前智能体"指示器,自动激活 + 手动切换 + `/agent <name>` 命令。
- 切换智能体只换 SystemPrompt/tool/Provider,不清 Persona/Episodic/WorkingMemory(与 `/new` 红线一致)。
- 预置 2–3 个开箱模板("学术论文 / 代码阅读 / 信息核查"),配置存 `chrome.storage.local`。

## 验收标准
- [ ] 配置页"智能体" Tab 完整 CRUD;预置模板可一键克隆。
- [ ] 自动激活 + 手动切换 + `/agent` 命令均可用。
- [ ] 切换不清 Persona/Episodic/WorkingMemory;默认通用助手不可删除。
- [ ] schema 字段 `type / trigger / outputChannel / defaultEnabled` 已预留(MVP 可只实装 interactive + main-stream,但字段必须先在)。
- [ ] README + ROADMAP 同步。
