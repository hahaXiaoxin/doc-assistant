# Persona 抽取边界显式化

- **ID**: ideas-005
- **状态**: proposed
- **优先级**: P0
- **来源**: `docs/backlog.md` · B-007-A
- **创建日期**: 2026-05-24

## 背景
`packages/agent/src/reflection/runner.ts` L236–L263 的 `runPersonaExtraction` prompt 只写"persona 是什么"(正例),没写"persona 不是什么"(反例);技能 / 工作流 / 领域知识 / 教学指令类内容没有被显式排除。aux 模型于是把"介绍 go 函数要注意顺序"判入 `subject='agent'` 行为方式,直达审核队列,造成噪音打扰用户(详见 bugs-003)。`packages/tools/src/definitions/remember-persona.ts` L52 的主 LLM tool description 同样缺反例。

## 提案
- 在 reflection prompt 与 tool description 中**新增"非 persona"反例段落**,共享同一份边界定义,至少覆盖:
  - 技能 / 领域知识类("介绍 go 函数要注意顺序")
  - 工作流 / 做事步骤类("写代码先写测试")
  - 一次性教学指令 / 具体任务指令("把这段翻译成英文")
  - 对页面内容的评价 / 情绪表达
- 离线 fixture 回归:技能/工作流类召回率 ≤ 设定上限、persona 类 ≥ 下限,覆盖 agent / user 两侧。

## 验收标准
- [ ] `runPersonaExtraction` system prompt 新增反例段落。
- [ ] `remember_persona` tool description 同步同一份反例。
- [ ] `packages/agent/src/__tests__/` 新增 fixture 回归。
- [ ] 不收窄 `PersonaSubject` 类型;不动已审核老 persona。
- [ ] `docs/requirements/v0.4.0-visible-memory.md` 追加注脚指向本条定版边界。
