# Persona 记忆自整理(写入侧相似度 + 合并工具)

- **ID**: ideas-006
- **状态**: proposed
- **优先级**: P1
- **来源**: `docs/backlog.md` · B-007-B
- **创建日期**: 2026-05-24

## 背景
`packages/agent/src/context/persona.ts` L38–L68 把 Persona 注入截断为 `agentTopK=10 / userTopK=8`(产品红线级硬上限)。dedupe 仅做严格字符串相等(L285–L287),"你说话要简洁" vs "回答保持简洁、不加客套"会各占一个 slot。结果:用户审核通过的长尾 persona 永远进不了 prompt — 审核了却"没用"(详见 bugs-004)。无脑提高 topK 会打爆 prompt + 带噪音;真正解法是"整合降数量",让长尾被语义归并到头部。

## 提案
- **写入侧 · 新增前相似度检查**:`addPersonaCandidate` 落 pending 前在同 subject 范围内向量/语义检索;命中阈值触发"merge"或"产合并建议"二选一(默认走合并建议进审核队列,记忆合并是单向不可逆的信任行为)。
- **合并批处理工具 `merge_personas(ids[])`**:供配置页"记忆浏览器"手动多选合并 + 反思 Job 后台扫相似簇产建议。
- **数据血统**:合并后 PersonaRecord 保留"合并来源 ID 列表",可回溯误合并修复。
- **相似度阈值产品参数化**(MVP 默认 0.85),归口到 ideas-015 AgentRuntimeConfig。
- **top-k 不动**;靠"整合降数量"让长尾自然进入 prompt。

## 验收标准
- [ ] 新增 candidate 时同义条目触发合并路径;hitCount/confidence 正确累积。
- [ ] 配置页"记忆浏览器"多选合并可用。
- [ ] 反思 Job 后台扫描产合并建议,落审核队列(默认不直接合并)。
- [ ] 合并后保留来源 ID 链路,UI 可回溯。
- [ ] 不动老数据;存量整理只在"高级用户手动触发"按钮下发起。
