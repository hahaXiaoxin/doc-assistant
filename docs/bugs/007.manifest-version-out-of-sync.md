# manifest 0.6.0-beta.1 与 ROADMAP / CHANGELOG 状态不一致

- **ID**: bugs-007
- **状态**: investigating
- **优先级**: P3
- **来源**: `docs/ROADMAP.md` 表格脚注 `[^manifest-0.6-beta]`(2026-05-06 归档登记)
- **创建日期**: 2026-05-24

## 现象
- `apps/extension/manifest.json` 的 `version` 字段为 `0.6.0-beta.1`。
- ROADMAP 表格中 v0.6 显示"规划中(待确认)";最近"已发布"的对外版本为 v0.5.0(后被 v0.6.0-beta.2 DeepSeek Provider 推翻 — 又是另一处不一致)。
- 简而言之:`manifest.version` 与文档中"最近已发版"的描述对不上,且 `0.6.0-beta.1` vs `0.6.0-beta.2`(CHANGELOG / requirements 实际有的是 beta.2)对不上。

## 复现
- `cat apps/extension/manifest.json | jq .version` → `0.6.0-beta.1`
- 对照 `docs/ROADMAP.md` 版本路线表与 `docs/CHANGELOG.md` 最近条目。

## 根因(待确认)
两种可能,需 developer / 维护者拍:
1. **发版抢号**:有人在跑 dev 时改了 manifest 但未发版未更新 ROADMAP。
2. **文档未同步**:实际已发 v0.6.0-beta.x 但 ROADMAP 行更新滞后。

## 期望行为
- `manifest.version` 与"最近已发布版本"在 ROADMAP / CHANGELOG / requirements 三处保持一致。
- 形成"发版前必同步 manifest + ROADMAP 表 + CHANGELOG"的清单(可作为 ideas-008 中 CONTRIBUTING.md 的一条 checklist)。

## 备注
- ROADMAP 脚注明确登记"本次归档不擅自改 manifest,也不擅自宣告 v0.6 发版,只是登记矛盾"。
- 影响面小(用户感知不强),但作为开源化打磨期的"卫生条目"应清掉。
