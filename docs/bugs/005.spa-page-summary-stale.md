# SPA 场景页面摘要过期未主动刷新

- **ID**: bugs-005
- **状态**: open
- **优先级**: P2
- **来源**: `docs/ROADMAP.md` · §6 "本期明确延后的能力清单"
- **创建日期**: 2026-05-24

## 现象
SPA 场景下页面内容动态变化(用户路由切换、内容懒加载),但 `getPageSummary()` 由 `useMemo([visible])` 缓存,只在面板显隐切换时重算。在 sidebar 一直保持打开 + SPA 页面持续变化的场景下,LLM 拿到的页面摘要可能与用户当前所看到的内容**不一致**。

## 复现
1. 在 SPA 应用(如 Notion / 飞书文档 / 知乎首页)打开 sidebar(visible=true)。
2. 在页面内做触发 SPA 路由 / 内容变化(不刷新整页)。
3. 与 LLM 对话"这页讲了什么" → LLM 引用的可能仍是上一段内容。
4. 关闭再打开 sidebar 触发重算,才会拿到新摘要。

## 根因
- `packages/ui/src/features/chat/ChatPanel.tsx` 的 `pageSummary = useMemo(() => getPageSummary(), [visible])` 是 TROUBLESHOOTING §5(输入卡顿 + 404 风暴)的缓存修复。
- 但缓存策略只关心 sidebar 显隐,没监听页面 DOM 变化、SPA 路由变化。
- v0.2.4 已加了 `hashchange` 监听清 SessionTopic,但 **`pageSummary` 本身没被刷**;`pushState/replaceState/popstate` 路径同样未联动。
- `send` 时 `buildInvokeContext` 会 `getPageSummary()` 即时取一次,所以发出的请求是新的;但 UI 卡片显示的是缓存值,且页面有重大变化时第一次请求也可能滞后。

## 期望行为
- SPA 路由 / 重大 DOM 变化时,主动失效 / 重算 `pageSummary`。
- 不能回退到"每次 render 都重算"(否则 §5 风暴复发)。
- 需要在"重算开销"与"内容时新性"之间设防抖 / 触发阈值。

## 备注
- ROADMAP 已登记为延后项(§6),修复时不要回退 §5 的缓存修复。
- 与 ideas-011(域名级 DSL 提取器)正交,但若 DSL 落地,DSL 提取器可作为更便宜的"重算"实现。
- 兼容 v0.2.4 `hashchange` 清 topic 的策略 — 不切 visit、只清 topic 的红线保留。
