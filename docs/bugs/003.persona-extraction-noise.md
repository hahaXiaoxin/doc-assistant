# Persona 抽取噪音 · 技能 / 工作流被误判进审核队列

- **ID**: bugs-003
- **状态**: open
- **优先级**: P0
- **来源**: `docs/backlog.md` · B-007-A
- **创建日期**: 2026-05-24

## 现象
用户原话:**"目前我在使用的过程中,可能我并不需要 doc-assistant 记住一些 persona 记忆,但是它依然会弹出进行审核,很多内容是跟技能相关的而非跟个人性格相关的。比如它会问:'给用户介绍 go 函数的时候要注意顺序'这种完全不属于一种个人性格、个人信息、个人方式的记忆,就不应该被加入 persona 之中,更不应该还丢给用户审核。"**

每次打开 sidebar 都能看见审核队列里塞满"非 persona"条目 → 拒掉一堆 → 信任崩溃 → 直接放弃审核。

## 复现
真机跑一段时间(几十轮对话,涉及代码 / 论文 / 技术解释),让反思 Job 跑几次。打开 PersonaReviewBanner,统计:技能 / 工作流 / 教学指令类条目占比。

## 根因
- `packages/agent/src/reflection/runner.ts` L236–L263 的 `runPersonaExtraction` system prompt **只写"persona 是什么"**(正例),没写"persona 不是什么"(反例);技能 / 工作流 / 领域知识 / 一次性教学指令类没有显式排除。
- aux 模型把"介绍 go 函数要注意顺序"判入 `subject='agent'` 行为方式。
- L297–L312 的 `addPersonaCandidate` 调用点中间**没有任何"该不该记"的闸门** — 抽取出什么直接落 `status: 'pending'`,噪音直达用户。
- `packages/tools/src/definitions/remember-persona.ts` L52 的主 LLM tool description 同样缺反例,主 LLM 也可能直接绕过反思链路写入 `confirmed` 级别噪音。

## 期望行为
- 技能 / 工作流 / 领域知识 / 教学指令类不应进入审核队列。
- persona(性格 / 偏好 / 长期行为风格)正常召回。
- 离线 fixture 量化:技能类召回率 ≤ 设定上限、persona 类 ≥ 下限。

- 建议搭车任意近期 prompt 相关迭代合入。
- 不收窄 `PersonaSubject` 类型;不动已审核老 persona。
